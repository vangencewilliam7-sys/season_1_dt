# Walkthrough: Workflow 5 - Student Monitoring & Risk Escalation

## Overview
This walkthrough covers the implementation and testing of the **Education Digital Twin: Monitoring & Risk Escalation** system. This feature moves the AI Tutor beyond a simple chatbot by performing "Deep Intelligence" analysis on student behavior.

## Core Features
1. **Support Triage classification**: Automatically flags students as `LOW`, `MEDIUM`, or `HIGH` risk based on behavioral patterns.
2. **Next Best Action (NBA) Engine**: Suggests concrete, expert interventions for human tutors.
3. **Deep Intelligence Metrics**:
   - **Curiosity Coefficient**: Measures conceptual depth vs. surface learning.
   - **Sentiment Trajectory**: Tracks mood changes over time.
   - **Help-Seeking Delay**: Detects "silent strugglers."
   - **Consistency Variance**: Flags habit shifts (potential burnout).

## How to Test

### 1. Backend Verification
Run the test script to see the Twin analyze various student profiles:
```bash
python backend/scratch/test_student_monitoring.py
```

### 2. Frontend Verification
1. Navigate to the **Education Skill** tab.
2. Open the **Student Monitoring Panel**.
3. Use the mock input fields to simulate a "Silent Struggler" (High login frequency, but low scores and high help-seeking delay).
4. Observe the **Risk Meter** and the **Next Best Action** card generated by the Twin.

## Key Files
- `backend/app/skills/functional/education/student_monitoring.py`: The brain of the monitoring system.
- `frontend/src/components/skills/education/StudentMonitoringPanel.js`: The "Tutor Intelligence" dashboard.
- `backend/app/skills/schemas/education.py`: Data structure for deep metrics.
